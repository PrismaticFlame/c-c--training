Phase I — C Foundations

Weeks 1–3 Pure C Beginner → Confident

Learn C before C++. C forces you to understand memory layout, pointers, and manual resource management with zero safety nets. These skills are what separate C++ programmers who understand the machine from those who cargo-cult smart pointers.

Week 01 The Machine Model: Types, Memory, Compilation ▶
Topics
Compilation pipeline
(pre → compile → link)
Primitive types & sizes
(sizeof, stdint.h)
Stack vs Heap mental model
Variables, scope, linkage
gcc/clang flags
(-Wall -Wextra -g -O2)
printf/scanf, I/O basics
Resources
📖 Beej's Guide to C (ch1–5) 🔍 cdecl.org
Exercises
Write a program that prints the size of every basic type on your machine.
Implement fizzbuzz using only for loops and printf.
Compile with -Wall -Wextra and eliminate every warning.
💡 Tip: Add -fsanitize=address,undefined to your compiler flags now and keep them for all of Phase I. Let AddressSanitizer yell at you — that's the fastest way to build pointer intuition.
Week 02 Pointers, Arrays, and the Address Space ▶
Topics
Pointer syntax & semantics
(*, &, ->)
Pointer arithmetic
Arrays decay to pointers
2D arrays & layout
const-correctness
NULL, dangling ptrs,
double-free
Resources
📖 Beej's Guide (ch6–8) 📖 Yale CS223 Notes
Exercises
Implement memcpy, strlen, and strrev from scratch.
Write a function that reverses an array in-place using only pointer arithmetic (no index operator).
Debug 3 intentional memory bugs using valgrind or ASan — document what each error message means.
💡 Tip: Draw the address space on paper for every exercise. Physical memory diagrams are more valuable than any tutorial at this stage.
Week 03 Functions, Structs, and Dynamic Allocation ▶
Topics
Function pointers
Structs & padding
malloc/calloc/realloc/free
Heap lifetime ownership
Linked list from scratch
Header files & .c/.h split
Exercises
Implement a singly-linked list with push, pop, find, and free_all.
Write a generic qsort wrapper using a function pointer comparator.
Build a dynamic array (like Python list) — auto-doubling capacity with realloc.
💡 Tip: Every malloc must have a clear owner. Start writing a comment above each heap allocation: "// Freed in: [function name]". This discipline maps directly to C++ RAII.
🎯
Phase I Milestone: You can write a correct, valgrind-clean C program from scratch with dynamic allocation, pointer arithmetic, and no memory leaks. You understand what the compiler actually generates.
⚙️

Phase II — C Mastery

Weeks 4–6 C · Systems Intermediate C

Push into systems-level C: bitwise ops, file I/O, process/signal handling, and build tooling. By the end you'll be comfortable reading kernel or HPC library source code.

Week 04 Bitwise Operations & Low-Level Types ▶
Topics
Bitwise &, |, ^, ~, <<, >>
Bit flags & masks
Unions & type punning
Endianness
Fixed-width types (uint32_t)
Bit fields in structs
Exercises
Implement a bitmask permission system (read/write/exec flags).
Write a function to count set bits (popcount) without using built-ins.
Use a union to inspect the bit layout of a float — connect to IEEE 754.
💡 Trans7 relevance: Stabilizer syndrome bits in QEC are literally bitstrings. Bit manipulation at this level is directly applicable to your decoder work.
Week 05 File I/O, Error Handling & POSIX Basics ▶
Topics

Exercises
Write a binary file serializer/deserializer for a struct with mixed types.
Re-implement wc -l (line counter) using only POSIX read() with a fixed buffer.
Use mmap to read a large file and measure throughput vs fread.
Week 06 Build Systems, Debugging & Profiling ▶
Topics
Makefiles (targets, deps, vars)
CMake basics
gdb / lldb workflows
valgrind --leak-check=full
perf & gprof basics
Static analysis (clang-tidy)
Exercises
Write a non-trivial Makefile with separate compile and link steps, and a clean target.
Use gdb to set a conditional breakpoint inside a loop — step through pointer arithmetic.
Profile your Week 3 linked list with perf stat and interpret cache-miss numbers.
💡 Tip: CMake is what you'll encounter in almost every real C/C++ project. Spend extra time here — even a shallow CMake understanding pays dividends fast.
🎯
Phase II Milestone: You can read a typical HPC or systems library in C, understand its Makefile/CMake, debug it with gdb, and profile it. You're ready to understand what C++ actually adds on top.

Phase III — C++ Core

Weeks 7–9 C++11/14 Foundational OOP & RAII

Now C++ makes sense because you know what it's abstracting. Focus on the "zero-cost abstraction" principle: understand exactly what the compiler generates for each C++ feature.

Week 07 Classes, RAII, and the Rule of Five ▶
Topics
Classes vs structs in C++
Constructors & destructors
RAII pattern
Rule of Three / Five / Zero
Copy & move semantics
this pointer
Resources
📖 learncpp.com (ch13–15) 📖 C++ Core Guidelines
Exercises
Re-implement your Week 3 dynamic array as a C++ class with proper RAII — no leaks possible.
Write a class that wraps a file descriptor and closes it in the destructor. Test that it's exception-safe.
Implement all five special member functions for a Matrix class — verify with a copy-and-modify test.
💡 Tip: Run your C++ under valgrind alongside the C version of the same task. Seeing identical output and zero leaks from RAII will cement why C++ exists.
Week 08 Templates, Operator Overloading & Namespaces ▶
Topics
Function & class templates
Template specialization
Operator overloading
Namespaces
inline & constexpr
Header-only libraries
Exercises
Write a generic Stack<T> template class backed by your dynamic array.
Implement a Vec3<T> with overloaded +, *, dot(), and operator[].
Use constexpr to compute Fibonacci at compile time — inspect the assembly output.
💡 Tip: Templates are compiled per-instantiation. Run g++ -c -S to see the generated assembly for Vec3<float> vs Vec3<double> — connects directly to your PyTorch dtype work.
Week 09 Inheritance, Polymorphism & Exceptions ▶
Topics

Exercises
Implement a Shape hierarchy (Circle, Rect, Triangle) with a virtual area() — measure vtable overhead with perf.

Write a class that provides the strong exception guarantee — document how RAII enables this.
🎯
Phase III Milestone: You can design a C++ class hierarchy, understand the cost of virtual dispatch, write exception-safe code, and use templates to avoid code duplication. You're at "knows C++" territory.
⚡

Phase IV — Modern C++ (C++17/20)

Weeks 10–12 C++17 · C++20 Production-Grade

The idioms that define contemporary C++: smart pointers, STL fluency, move semantics depth, structured concurrency, and the features that make C++ competitive with Rust for systems work.

Week 10 Smart Pointers & Modern Memory Management ▶
Topics
unique_ptr & ownership
shared_ptr & weak_ptr
make_unique / make_shared
Move semantics deep dive
std::move vs std::forward
Perfect forwarding
Exercises
Port your Week 3 linked list to use unique_ptr<Node> — notice what disappears.
Build a tree with shared_ptr children and weak_ptr parent links — avoid cycles.
Write a factory function using perfect forwarding — confirm zero unnecessary copies with a copy-counting wrapper type.
💡 Tip: unique_ptr has zero runtime overhead over a raw pointer. Verify this by looking at the disassembly. This is the "zero-cost abstraction" promise made concrete.
Week 11 STL Fluency & Algorithm Library ▶
Topics

unordered_map, set

(sort, find, transform…)
Lambdas & closures

Structured bindings
Exercises
Solve 10 LeetCode mediums using only STL containers and <algorithm> — no manual loops for sorting/searching.
Implement a word-frequency counter that uses unordered_map and is sorted via std::sort on a vector of pairs.
Rewrite your Vec3 operations using C++20 ranges — compare readability to raw loops.
Week 12 Concurrency: Threads, Atomics & Coroutines ▶
Topics
std::thread & jthread
mutex, lock_guard, unique_lock
condition_variable
std::atomic
Memory ordering basics
C++20 coroutines (intro)
Exercises
Implement a thread-safe queue using mutex + condition_variable.
Write a parallel map using a thread pool — compare wall time to single-threaded on a compute-heavy task.
Reproduce a data race with TSan (-fsanitize=thread), then fix it with std::atomic.
💡 HPC relevance: Your SLURM/DDP work is multi-process, but C++ concurrency is intra-process multi-threading. Understanding both lets you reason about the full memory hierarchy on TinkerCliffs.
🎯
Phase IV Milestone: You write modern C++ by default — smart pointers, STL algorithms, lambdas, move semantics. You can write and debug multithreaded C++ code and understand memory ordering. This is intermediate.

Phase V — Capstone Projects

Weeks 13–14 C++ · Systems Consolidation

Pick one or two of these projects that align with your actual goals. A finished project beats another tutorial. These are sized for 1–2 weeks of focused work each.

Project A Tensor Library (ML Infra track) ▶

Build a minimal N-dimensional tensor with contiguous memory, strided views, broadcasting rules, and BLAS-backed matmul. Directly reinforces Trans7/PyTorch internals knowledge.

Must implement
Template Tensor<T, N> with shape, strides, and a heap-allocated data buffer.
Slice / view semantics — non-owning subview with offset strides.
Element-wise ops via expression templates (lazy evaluation).
matmul backed by cblas_sgemm from OpenBLAS.
Project B HTTP/1.1 Server from Sockets (Systems track) ▶

Build a concurrent HTTP/1.1 server from POSIX sockets. No libraries. Forces mastery of async I/O, thread pooling, and protocol parsing — the same layer your FastAPI/Kafka streaming engine sits on.

Must implement
TCP socket server with accept loop and thread-per-connection model.
HTTP/1.1 request parser (method, path, headers, body).
Router with GET/POST handler registration via lambdas.
Thread pool with work queue — replace thread-per-connection.
🏁
Final Milestone: You have a non-trivial, working C++ project on GitHub. You can read production C++ codebases (libtorch, BLAS, LLVM internals) and reason about their design decisions. You are intermediate.
